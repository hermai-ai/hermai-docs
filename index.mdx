Hermai is a schema registry where developers contribute website API schemas and AI agents consume them. Point an agent at any domain in the registry and get back the full list of callable endpoints — URLs, methods, headers, parameters — ready to execute. No browser, no HTML parsing, no wasted tokens.

Think of it as **HuggingFace, but for website APIs**.

## How it works

<CardGroup cols={2}>
  <Card title="Discover with the CLI" icon="terminal">
    Run `hermai discover` locally to find every API endpoint a website exposes. The CLI handles browser automation, LLM analysis, and schema generation — you just give it a URL.
  </Card>
  <Card title="Look up from the registry" icon="magnifying-glass">
    Call `GET /v1/catalog/{domain}` to get all known endpoints for a site. The platform serves schemas contributed by the community — instant, structured, no discovery cost.
  </Card>
</CardGroup>

## Open source

The [hermai-cli](https://github.com/hermai-ai/hermai-cli) is open source under AGPL-3.0. You can run the full discovery engine locally without an account. The platform registry is the community layer on top — browse, search, and consume schemas that others have contributed.
