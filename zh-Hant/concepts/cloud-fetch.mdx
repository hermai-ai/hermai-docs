Catalog 告訴 agent「該怎麼呼叫網站」;Cloud fetch 則是「直接幫你呼叫」。呼叫單一 Hermai 端點,我們會代為發出對上游網站的請求、處理 anti-bot 挑戰、管理硬站的 session 與簽章請求,並回傳乾淨、已投射成 schema 欄位的 JSON。

你不想自己寫 proxy / cookie / 簽章程式碼時就用它,或當網站是那些不會回應資料中心 IP 純 HTTPS 呼叫的站點時就更需要它。

<Tip>
  **AI agent 在讀這頁?** 最精簡的流程是:

  1. `POST https://fetch.hermai.ai/v1/fetch`,header 加 `Authorization: Bearer hm_sk_…`
  2. Body 形態:`{"site": "<site>", "endpoint": "<endpoint>", "params": {…}}`
  3. 要查 `<site>` 與 `<endpoint>` 的名字,呼叫 `GET https://api.hermai.ai/v1/catalog/<domain>`,header 加 `X-Hermai-Intent: <一句話說明為何呼叫>`(20 字以上、5 個字以上)
  4. 回應是 `{"data": {…乾淨投射過的 JSON…}, "site": …, "endpoint": …}` — 讀 `data` 就好

  這個流程對 registry 內任何網站都成立,包含 anti-bot 硬站(x.com、booking.com、instagram.com、tiktok.com)。
</Tip>

## 5 分鐘第一次呼叫

從零開始到看到乾淨 JSON 的完整流程。

<Steps>
  <Step title="申請 API key">
    在 [hermai.ai/register](https://hermai.ai/register) 註冊、開啟 dashboard,並在 **API Keys** 下建立一支金鑰。複製下來(以 `hm_sk_…` 開頭,只會顯示一次)。

    ```bash
    export HERMAI_KEY=hm_sk_…
    ```
  </Step>
  <Step title="從 catalog 找一個網站">
    Catalog 會告訴你 Hermai 知道哪些網站,以及每個網站開放哪些 endpoint。先挑一個你想用的網域:

    ```bash
    curl -H "Authorization: Bearer $HERMAI_KEY" \
         -H "X-Hermai-Intent: 為開發者研究 agent 查詢 github 使用者資料" \
         "https://api.hermai.ai/v1/catalog/github.com" | head -c 800
    ```

    會拿到一份 endpoint 列表。每個 endpoint 都有 `name`、`method`、`url`,以及視情況的 `params` 列表。挑一個 — 例如 `GetUser`。
  </Step>
  <Step title="執行 cloud-fetch 呼叫">
    把 endpoint 交給 Hermai:

    ```bash
    curl -X POST "https://fetch.hermai.ai/v1/fetch" \
         -H "Authorization: Bearer $HERMAI_KEY" \
         -H "Content-Type: application/json" \
         -d '{"site":"github.com","endpoint":"GetUser","params":{"username":"torvalds"}}'
    ```

    回應(節錄):

    ```json
    {
      "data": { "login": "torvalds", "name": "Linus Torvalds", "followers": 302438, "…": "…" },
      "site": "github.com",
      "endpoint": "GetUser"
    }
    ```
  </Step>
  <Step title="試試硬站">
    同一個形態、更硬的網站。自己直連這個會 500:

    ```bash
    curl -X POST "https://fetch.hermai.ai/v1/fetch" \
         -H "Authorization: Bearer $HERMAI_KEY" \
         -H "Content-Type: application/json" \
         -d '{
           "site": "x.com",
           "endpoint": "user_profile",
           "params": {
             "variables": "{\"screen_name\":\"jack\",\"withSafetyModeUserFields\":true}",
             "features": "{\"hidden_profile_likes_enabled\":true}"
           }
         }'
    ```

    約 1.5 秒回傳完整 profile。Hermai 替你打點了 session 與簽章。
  </Step>
</Steps>

你剛剛做了兩次 cloud-fetch 呼叫。同樣的模式適用[registry](https://hermai.ai/schemas) 內每個網站。每次成功扣 1 credit;起始餘額夠你跑上千次。

## 兩種呼叫方式

Cloud fetch 有兩種請求形態。兩者都用你的 `hm_sk_…` 金鑰驗證,並各自**每次成功回應扣 1 個 credit**。

<CardGroup cols={2}>
  <Card title="用 URL" icon="link">
    `POST https://api.hermai.ai/v1/fetch`

    Body:`{"url": "https://…"}`

    最適合已經有 URL 的公開 API。
  </Card>
  <Card title="用名稱" icon="bullseye">
    `POST https://fetch.hermai.ai/v1/fetch`

    Body:`{"site": "...", "endpoint": "...", "params": {…}}`

    最適合 anti-bot 網站、需要簽章或登入態的端點。
  </Card>
</CardGroup>

## 何時用哪種

| 網站類型 | 範例 | 用哪個 |
|---|---|---|
| 公開 JSON API、無 anti-bot | github.com、bitbucket.org、coinbase.com、weather.gov、npmjs.com、nasa.gov | **用 URL** — 請求形態更簡單 |
| Anti-bot / 簽章 / cookie 綁定 | x.com、booking.com、threads.com、instagram.com、tiktok.com、airbnb.com | **用名稱** — 必須;在這些網站「用 URL」會回傳空資料或 5xx |
| Registry 內任何網站,但你手邊沒有確切 URL | 任何網站 | **用名稱** — agent 普遍偏好;不論網站差異都是同一種請求形態 |

不確定時,先在 catalog 查一下(`GET /v1/catalog/{domain}`)。Catalog 回應會告訴你有哪些端點以及各自需要什麼參數 — 這就是「用名稱」呼叫所需的全部資訊。

## 用 URL 呼叫

<CodeGroup>
```bash curl
curl -X POST https://api.hermai.ai/v1/fetch \
  -H "Authorization: Bearer hm_sk_…" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://api.github.com/users/torvalds"}'
```

```python Python
import requests

resp = requests.post(
    "https://api.hermai.ai/v1/fetch",
    headers={"Authorization": "Bearer hm_sk_…"},
    json={"url": "https://api.github.com/users/torvalds"},
)
data = resp.json()
print(data["data"]["login"], "—", data["data"]["name"])
```

```js Node

```

```go Go

    "bytes"
    "encoding/json"
    "fmt"
    "net/http"
)

func main() {
    body, _ := json.Marshal(map[string]string{

    req, _ := http.NewRequest("POST", "https://api.hermai.ai/v1/fetch", bytes.NewReader(body))

    var out struct {
        Data map[string]any `json:"data"`

    json.NewDecoder(resp.Body).Decode(&out)
    fmt.Println(out.Data["login"], "—", out.Data["name"])
}
```

Hermai 替你打點 anti-bot 細節 — 對的 cookie、對的 TLS profile、需要時的請求簽章、上游推回時的重試。Agent 只需要說出它要什麼。

### 怎麼找到正確的 `endpoint` 與 `params`

1. 先在 catalog 查站:

   ```bash
   curl -H "Authorization: Bearer hm_sk_…" \
        -H "X-Hermai-Intent: 在 X 上搜尋與 Zeabur 相關的貼文,彙整開發者回饋" \
        "https://api.hermai.ai/v1/catalog/x.com"
   ```

2. 從回應中按 `name` 挑一個 endpoint。
3. 看該 endpoint 的 `params` 列表,確認必要欄位。
4. 用 `{"site":"x.com","endpoint":"<name>","params":{…}}` 呼叫 `fetch.hermai.ai/v1/fetch`。

## 範例:在 X 搜尋「Zeabur」

```bash
curl -X POST https://fetch.hermai.ai/v1/fetch \
  -H "Authorization: Bearer hm_sk_…" \
  -H "Content-Type: application/json" \
  -d '{
    "site": "x.com",
    "endpoint": "search",
    "params": {
      "variables": "{\"rawQuery\":\"Zeabur\",\"count\":20,\"querySource\":\"typed_query\",\"product\":\"Latest\"}",
      "features": "{…}",
      "referer_query": "Zeabur"
    }
  }'
```

回應會帶回搜尋時間軸 — 20 則貼文,含作者、內文、按讚/轉推數、時間戳記。你這端不用瀏覽器、也不用搞 cookie。

## Agent 的程式化探索

多數 agent 一開始並不知道哪些網站該用「用 URL」、哪些該用「用名稱」。最穩當的演算法是「永遠走 catalog」 — 它會告訴你所有需要的資訊:

```python
import requests, urllib.parse

def hermai_fetch(domain, intent, key, endpoint=None, params=None):
    """一站式包裝:先在 catalog 查站,再執行 cloud-fetch。"""
    headers = {"Authorization": f"Bearer {key}", "X-Hermai-Intent": intent}

    # 1. 查 catalog
    catalog = requests.get(
        f"https://api.hermai.ai/v1/catalog/{urllib.parse.quote(domain)}",
        headers=headers,
    ).json()

    if not catalog.get("endpoints"):
        raise RuntimeError(f"registry 內沒有 {domain} 的 schema")

    # 2. 挑 endpoint(呼叫者指定名字,否則預設第一個讀取端點)
    ep = next((e for e in catalog["endpoints"] if e["name"] == endpoint), None) \
         if endpoint else catalog["endpoints"][0]

    # 3. Schema-mode 是最安全的預設 — 對每種網站都有效
    resp = requests.post(
        "https://fetch.hermai.ai/v1/fetch",
        headers={"Authorization": f"Bearer {key}"},
        json={"site": domain, "endpoint": ep["name"], "params": params or {}},
    )
    return resp.json()


# 用法
print(hermai_fetch("github.com", "為程式碼研究 agent 取得 torvalds 的個人檔案",
                   "hm_sk_…", endpoint="GetUser", params={"username": "torvalds"}))

print(hermai_fetch("x.com", "為了豐富聯絡卡而取得 jack 的個人檔案",
                   "hm_sk_…", endpoint="user_profile",
                   params={"variables": '{"screen_name":"jack"}', "features": "{}"}))
```

**Agent 的經驗法則**:不確定時就用 **schema-mode**(`fetch.hermai.ai`)。它是唯一適用所有網站類型的單一形態 — 公開 API、anti-bot 硬站、簽章端點全包。URL-mode 只是當你手邊已經有 URL 字串時的方便寫法。

## 可能會看到的錯誤

| 狀態碼 | 意思 | 怎麼處理 |
|---|---|---|
| `401 invalid or revoked API key` | 金鑰錯誤或已撤銷 | 檢查 `Authorization` header |
| `402 INSUFFICIENT_CREDITS` | Credit 不足 | 在 dashboard 加值 |
| `404 site not found` | 該網站尚未在 registry | 用 CLI 貢獻一份 — 見 [Schemas](/zh-Hant/concepts/schemas) |
| `404 endpoint not found on schema` | endpoint 名稱錯誤 | 回 catalog 確認可用的 `name` |
| `429 too many requests` | Workspace rate limit 用完 | 退避後重試;回應含 `Retry-After` |
| `503 pool_empty` | 該網站需要託管 session,且池暫時為空 | 數秒後重試 — Hermai 會自動補池 |

## 怎麼跟其他方案比

- **vs. 自己寫爬蟲**:不必處理 proxy、瀏覽器、cookie 與簽章。一個 HTTPS 呼叫,回乾淨 JSON。
- **vs. 先 catalog 再自己呼叫**:catalog-then-call 對純 HTTPS 公開 API 是免費的。Cloud fetch 每次成功扣 1 credit,但能搞定會擋直連的網站,且不論網站差異都是同一個介面。
- **vs. 在本機跑 CLI**:CLI 跑在你的機器上 — 適合想用自己的 cookie 或登入態瀏覽器 session。Cloud fetch 跑在我們的機器上 — 適合本地零設定,或你在做伺服器端 bot。

## Credits

Cloud fetch **每次成功回應扣 1 個 credit**。失敗(我們重試後仍 4xx/5xx,或 schema-defect)不扣 credit — 預留會自動釋放。如何累積與加值請見 [Credits & usage](/zh-Hant/concepts/credits-and-usage)。

## Catalog 查詢仍免費

`GET /v1/catalog/{domain}` 與 `GET /v1/schemas/{site}/package` 不計入 credit。Cloud fetch 是唯一會計費的介面。這讓 agent 可以先自由探索 registry,再決定要不要對某個端點呼叫 cloud-fetch。
