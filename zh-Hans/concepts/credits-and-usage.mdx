## 速率限制

所有 API 请求都有速率限制。限制取决于是否已验证:

| 层级 | 每小时 | 每天 | 身份跟踪方式 |
|------|-------|------|------------|
| 匿名(无 API key) | 5 | — | 按 IP 地址 |
| 已注册(免费) | 50 | 500 | 按用户账号 |

超过限制会收到 `429 Too Many Requests`。每个响应的 `X-Rate-Limit-Remaining` header 会告诉 agent 当前窗口还剩多少次请求。

获取 API key 免费 — 到 [hermai.ai/register](https://hermai.ai/register) 注册并在 dashboard 创建。

## Credits

Credits 通过贡献与社区活动赚取,用于 [cloud fetch](/zh-Hans/concepts/cloud-fetch) 调用。

### 赚取

| 行为 | Credits |
|------|---------|
| 第一次发起请求 | +50 |
| 在 GitHub 上 star [hermai-cli](https://github.com/hermai-ai/hermai-cli) | +100 |
| 推荐朋友 | 每位受邀 +100 |
| 贡献 schema | 每个网站 +50 |

### 消耗

| 调用 | Credits |
|------|---------|
| `POST https://api.hermai.ai/v1/fetch`(cloud fetch 用 URL) | 每次成功 1 |
| `POST https://fetch.hermai.ai/v1/fetch`(cloud fetch 用名称) | 每次成功 1 |
| 失败的 cloud-fetch 调用(重试后仍 4xx/5xx) | 0 — 预留会自动释放 |

Catalog 查询、registry 浏览与 schema 推送都免费 — 不会消耗 credits。

## 哪些算一次请求?

以下端点每次调用都计入速率限制:

- `GET /v1/catalog/{domain}`(catalog 查询)
- `GET /v1/schemas/{site}/package`(schema 下载)
- `POST /v1/fetch`(在 `api.hermai.ai` 或 `fetch.hermai.ai` 上)

以下**不**计入速率限制:

- `GET /v1/schemas`(浏览/搜索 registry)
- `GET /v1/schemas/{site}`(公开 schema 卡片)
- `GET /v1/trending`、`GET /v1/categories`(探索接口)
- `POST /v1/schemas`(推送 schema)
- `GET /v1/health`

## 用量 dashboard

你的 [dashboard](https://hermai.ai/dashboard) 会显示:

- **今日请求数** — 累计调用次数
- **Schema 命中率** — 查询命中已缓存 schema 的比例
- **平均延迟** — 请求的平均响应时间
- **已获得 credits** — 累计的 credits 余额
- **活动记录** — 每条请求的 URL、状态、延迟、时间戳
