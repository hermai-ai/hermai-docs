## 速率限制

所有 API 請求都有速率限制。限制視是否已驗證而定:

| 層級 | 每小時 | 每天 | 身份追蹤方式 |
|------|-------|------|------------|
| 匿名(無 API key) | 5 | — | 依 IP 位址 |
| 已註冊(免費) | 50 | 500 | 依使用者帳號 |

超過限制會收到 `429 Too Many Requests`。每個回應的 `X-Rate-Limit-Remaining` header 會告訴 agent 目前視窗還剩幾次請求。

取得 API key 免費 — 到 [hermai.ai/register](https://hermai.ai/register) 註冊並在 dashboard 建立。

## Credits

Credits 透過貢獻與社群活動賺取,並用在 [cloud fetch](/zh-Hant/concepts/cloud-fetch) 呼叫上。

### 賺取

| 行為 | Credits |
|------|---------|
| 第一次發送請求 | +50 |
| 在 GitHub 上 star [hermai-cli](https://github.com/hermai-ai/hermai-cli) | +100 |
| 推薦朋友 | 每位受邀 +100 |
| 貢獻 schema | 每個網站 +50 |

### 消耗

| 呼叫 | Credits |
|------|---------|
| `POST https://api.hermai.ai/v1/fetch`(cloud fetch 用 URL) | 每次成功 1 |
| `POST https://fetch.hermai.ai/v1/fetch`(cloud fetch 用名稱) | 每次成功 1 |
| 失敗的 cloud-fetch 呼叫(重試後仍 4xx/5xx) | 0 — 預留會自動釋放 |

Catalog 查詢、registry 瀏覽與 schema 推送都免費 — 不會消耗 credits。

## 哪些算一次請求?

以下端點每次呼叫都計入速率限制:

- `GET /v1/catalog/{domain}`(catalog 查詢)
- `GET /v1/schemas/{site}/package`(schema 下載)
- `POST /v1/fetch`(在 `api.hermai.ai` 或 `fetch.hermai.ai` 上)

以下**不**計入速率限制:

- `GET /v1/schemas`(瀏覽/搜尋 registry)
- `GET /v1/schemas/{site}`(公開 schema 卡片)
- `GET /v1/trending`、`GET /v1/categories`(探索介面)
- `POST /v1/schemas`(推送 schema)
- `GET /v1/health`

## 用量 dashboard

你的 [dashboard](https://hermai.ai/dashboard) 會顯示:

- **今日請求數** — 累計呼叫次數
- **Schema 命中率** — 查詢命中已快取 schema 的比例
- **平均延遲** — 請求的平均回應時間
- **已獲得 credits** — 累計的 credits 餘額
- **活動紀錄** — 每筆請求的 URL、狀態、延遲、時間戳

## 目前的免費方案

免費方案涵蓋上方速率限制範圍內的所有功能,並包含你從貢獻與社群活動賺到的 credits。註冊不需信用卡 — 註冊後貢獻一個 schema,就可以馬上開始呼叫 cloud-fetch。針對更高流量與專屬容量的付費方案即將推出,最新定價請見[dashboard](https://hermai.ai/dashboard)。
