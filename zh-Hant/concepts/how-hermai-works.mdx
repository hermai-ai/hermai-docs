## 兩端分工

Hermai 有兩端:

1. **CLI**([hermai-cli](https://github.com/hermai-ai/hermai-cli),開源)— 在你的機器上執行。貢獻者用它來檢視網站、撰寫描述端點的 schema,然後把 schema 推送到 registry。

2. **Platform**(api.hermai.ai + fetch.hermai.ai)— schema registry **加上**託管執行介面。Registry 儲存並提供貢獻者推送的 schema;agent 在執行期查詢以取得 registry 中任何網域的即時、結構化端點資料。接著 agent 可以選擇用 catalog 的資料直接呼叫上游網站,或讓 Hermai 透過 [cloud fetch](/zh-Hant/concepts/cloud-fetch) 代為執行。

```

    │                                  │                               │
    │  discovery toolkit (local)       │                               │
    │  hermai registry push            │                               │
    │──────────────────────────────>   │                               │
    │                                  │   GET /v1/catalog/{domain}    │
    │                                  │ <────────────────────────────│

    │                                  │ ────────────────────────────> │

    │                                  │ ────────────────────────────> │
```

## CLI 做什麼

CLI 提供一組確定性的工具組 — 每個子指令做一件事,並輸出可供下一步處理的 JSON:

- **`hermai detect`** — 分類網站平台(Shopify、WordPress、Next.js 等)並偵測 anti-bot 系統。
- **`hermai wellknown`** — 探測 `robots.txt`、sitemap、RSS feed 與 GraphQL 端點。
- **`hermai probe --body | hermai extract`** — 從頁面擷取內嵌資料。可辨識 13 種 SSR/hydration 模式(`__NEXT_DATA__`、`ytInitialData`、`__APOLLO_STATE__`、`SIGI_STATE` 等)。
- **`hermai intercept`** — 啟動瀏覽器並捕獲真實 XHR 流量,適用於動態功能(搜尋、購物車、篩選)。
- **`hermai introspect`** — 取得 GraphQL schema。
- **`hermai session bootstrap`** — 為 anti-bot 網站預熱瀏覽器 session。

貢獻者依照探索結果撰寫 schema JSON 檔,並以 `hermai registry push` 推送。不需要 LLM key 或託管服務。

## Platform 做什麼

Platform 有兩個介面:

**Catalog(唯讀、免費):**

- **`GET /v1/catalog/{domain}`** — 回傳該網域所有端點與 action。主要的 agent 查詢介面,需要[intent](/zh-Hant/concepts/catalog#intent)說明用途。
- **`GET /v1/schemas`** — 瀏覽並搜尋整個 registry。
- **`POST /v1/schemas`** — 推送新 schema。會依[intent 分類](/zh-Hant/concepts/catalog#intent-taxonomy)進行驗證並立即發布。

**Cloud fetch(託管執行、按次計費):**

- **`POST https://api.hermai.ai/v1/fetch`** 加 `{"url":"…"}` — Hermai 代你呼叫該 URL,並依匹配到的 schema 投射回應。最適合本來就吃純 HTTPS 的公開 API。
- **`POST https://fetch.hermai.ai/v1/fetch`** 加 `{"site":"…","endpoint":"…","params":{…}}` — Hermai 處理 anti-bot、session、簽章請求等硬站需求(X/Twitter、Booking、Threads、Instagram、TikTok 等)。同樣用 Bearer 金鑰驗證。

Cloud fetch 可選用 — 對沒有 anti-bot 的網站,agent 也可以拿 catalog 資料自己呼叫。完整參考見 [Cloud fetch](/zh-Hant/concepts/cloud-fetch)。

## 為什麼要這樣分工?

探索很貴(瀏覽器 + LLM + 時間)。查詢很便宜(資料庫讀取)。執行則介於中間,視網站而定。把它們分開後:

- **貢獻者**只需在自己機器上,本地付一次探索成本
- **之後的每一個 agent**都能免費取得即時結構化 catalog 資料
- **只想拿到乾淨 JSON 的 agent**呼叫 cloud fetch,免去自己寫 HTTP / proxy / cookie / 簽章程式
- **Platform**各層獨立擴展 — registry、catalog、execution 是分開的服務
- **不鎖死平台** — CLI 是開源(AGPL-3.0)且可離線運作;Cloud fetch 是按次選用
